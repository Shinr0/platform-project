02 / Stack technique

Technologies

Chaque composant a un role precis dans la chaine de provisioning. Aucun acces direct au cloud n'est necessaire pour les utilisateurs.

K8s

Kubernetes (Kind)

Cluster local servant de control plane. Heberge Crossplane et ArgoCD. Toute la logique d'orchestration passe par l'API Kubernetes.

XP

Crossplane

Transforme Kubernetes en control plane cloud universel. Definit des APIs custom (XRD) qui mappent vers des ressources GCP reelles via des Compositions.

CD

ArgoCD

Operateur GitOps. Surveille le dossier crossplane/claims/ du repo Git et synchronise automatiquement tout changement vers le cluster.

CI

GitHub Actions

Genere les manifests Crossplane (Claims YAML) a partir des inputs utilisateur et les commit dans le repo. Declenchement via workflow_dispatch.

GCP

Google Cloud Platform

Cloud cible. Le provider Upbound Crossplane communique avec l'API GCP pour creer/supprimer les ressources (Compute Engine, Disk, Address).

UI

Frontend Portal

Interface web statique (HTML/CSS/JS). Formulaire de creation de VM qui trigger le workflow GitHub Actions via l'API REST.

04 / Crossplane API

Custom Resource Definition

Crossplane expose une API Kubernetes custom VirtualMachine qui abstrait la complexite GCP derriere une interface simple et declarative.

claim-example.yaml
apiVersion: compute.platform.local/v1alpha1 kind: VirtualMachine metadata: name: my-dev-server namespace: default spec: parameters: vmName: my-dev-server machineType: e2-small # default: e2-micro zone: europe-west1-b diskSizeGb: 30 # default: 20 imageFamily: debian-12 spot: true # VM preemptible (cheaper) staticIp: true # external static IP dataDiskSizeGb: 100 # additional data disk
Parametre Type Defaut Description
vmName string - Nom de l'instance Compute Engine requis
machineType string e2-micro Type de machine GCP optionnel
zone string europe-west1-b Zone GCP de deploiement optionnel
diskSizeGb integer 20 Taille du disque de boot (GB) optionnel
imageFamily string debian-12 Famille d'image OS optionnel
spot boolean false VM Spot/Preemptible (moins cher, terminable par GCP) optionnel
staticIp boolean false Attacher une IP externe statique optionnel
dataDiskSizeGb integer 0 Disque de donnees supplementaire (0 = aucun) optionnel
05 / Features

Capacites de la plateforme

Ce que la plateforme permet aujourd'hui et les principes architecturaux qui la gouvernent.

Self-service provisioning

Les developpeurs creent et suppriment des VMs sans intervention Ops, via un portail web ou directement depuis GitHub Actions.

GitOps natif

L'etat desire de l'infrastructure est versionne dans Git. Chaque changement est tracable, auditable et reversible via un simple git revert.

Reconciliation continue

Crossplane surveille en permanence l'etat reel sur GCP et le reconcilie avec l'etat declare dans Kubernetes. Un drift est automatiquement corrige.

VM Spot / Preemptible

Option pour provisionner des VMs preemptibles a cout reduit, ideales pour les workloads tolerants aux interruptions.

IP statique

Reservation automatique d'une adresse IP externe statique quand l'option est activee, avec derivation auto de la region depuis la zone.

Disque supplementaire

Ajout optionnel d'un disque de donnees pd-standard, provisionne et attache automatiquement a l'instance.

Abstraction via XRD

Les utilisateurs ne manipulent qu'une API simple (VirtualMachine). La complexite GCP (Instance, Disk, Address) est masquee par la Composition.

Idempotent et reproductible

Le setup complet (cluster, Crossplane, ArgoCD, providers) est automatise via un script idempotent. Reproductible en quelques minutes.
